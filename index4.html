Frameworks

Un framework es una herramienta de desarrollo web que se define como una aplicación o conjunto de módulos que permiten un desarrollo más ágil de aplicaciones, mediante a la aportación de algunas librerías y/o funcionalidades ya creadas.

Una ventaja del uso de framework es que el diseñador se puede centrar completamente en algún problema que desee resolver y no en la implementación de funcionalidades que normalmente son de uso común y ya están resueltas por alguien más.

Modulo flexbox

Es candidato a recomendación del W3C a partir de octubre del 2017.

Tiene como objetivo proporcionar una manera más eficiente la distribución del espacio entre los elementos en un contenedor, incluso cuando su tamaño es desconocido. La idea principal es otorgar al contenedor la capacidad de modificar los lados y elementos para llenar mejor el espacio.

Existen propiedades que se aplican directamente al contenedor y otras que se le aplican a los hijos. Además de que cuenta con:

  • Soporte del navegador.
  • display:flex;sintaxis reciente de la especificación.
  • display :flexbox, sintaxis no oficial del 2011.
  • display:box; sintaxis antigua de 2009.

Propiedades del elemento contenedor.

Se modifica el elemento padre o elemento contenedor y conforme a este, los elementos hijos serán afectos.

Propiedades para los elementos flexibles.

1. Order, mostrara a los elementos con flujo normal de la página, pero también podemos controlar el orden de los elementos. Por default order se encuentra en 0, pero puede ser modificado con números positivos y negativos.

2. flex-grow, define la capacidad para que un ítem crezca si es necesario, acepta un valor entero, además indica la cantidad de espacio disponible dentro del contenedor. No le es válido los números negativos, y por default este se encuentra en 0.

Además, aumentara las unidades que se le hayan establecido correspondientemente al espacio disponible que existe en el elemento contenedor. para ello se calcula de la siguiente manera:

Unidad total= Espacio disponible/La suma del flex-grow de los ítems.

5. flex-shrink, especifica la capacidad de un elemento para reducir en relación con el resto de los elementos flexibles del contenedor, además por default este se encuentra en 1 unidad.

Al contrario del anterior, este especifica el comportamiento de los ítems cuando el tamaño del contenedor es menor al de los ítems que lo componen.

Para ello se calcula de la siguiente manera:

unidad de reducción=espacio faltante/suma de flex-shrink

4. flex-basis, define el tamaño predeterminado de un elemento antes de distribuir el espacio restante, por ejemplo, una longitud.

5. flex-basis:numero; estable hasta que longitud los elementos distribuirá su tamaño referente a una medida limitante.

6. flex, es una abreviatura de flex-grow,flex-shrink y flex, basis combinadas en una sola, sus valores predeterminados son 1,0, auto.

7. align-self, permite que la alineación predeterminada se anule para los elementos flex.

Mobil first

Una investigación realizada por la GSMA afirma que el uso del internet aumentará dispositivos móviles crecerá a más del 50%.

Estar conectados a internet se a convertido en una actividad de nuestro día a día, y poco a poco empezamos a dejar de usar la computadora para trasladarnos a los dispositivos móviles.

En 2015, Google estaba hablando sobre esta tendencia y priorizo los sitios web que son mobile friendly, en los que los resultados de la búsqueda.

Luke Wroblewski en 2011 desarrollo el concepto de mobile first, cuando publico el libro Mobile First, en el cual propone que primero se debe crear un diseño web pensado primero para un dispositivo móvil, para después ajustarlo al tamaño de una pantalla de computadora.

Para todo esto debemos tener en cuenta que todo lo que no es esencial o relevante debe de ser descartado.

Lo primero para empezar un diseño web es que se debe tener claro el contenido básico y sepáralo del contenido sobrante y prescindible.

Las decisiones deben tomarse de acuerdo con las funciones del usuario.

Si el espacio realmente es pequeño, siempre puede darse la opción de habilitar un menú de barra con las opciones que no se puedan incluir en la pantalla.

Algunos de los repor que presenta Mobile first son:

  • El tamaño de la pantalla reducido
  • La conexión móvil limitada
  • El comportamiento de los usuarios
  • Las pantallas 'touch', etc.

Patrones de diseño

Los patrones de diseño web adaptables evolucionan rápidamente, pero existen varios patrones establecidos que funcionan bien en los diferentes equipos de escritorio y dispositivos móviles.

Categorías de patrones:

  • mostly fluid. El origen del nombre "mayormente liquido" es debido a que la estructura principal de los layouts no cambia realmente hasta dimensiones obiles donde se aplica la caída de columna, el patrón consiste, principalmente, en una cuadricula fluida.
  • column drop. El tamaño total de los elementos está a disocian de mantenerse constante, se adapta a distintos tamaños de la pantalla. en caso de contener varias columnas.
  • layout shifter Es el más adaptable, ya que posee varios puntos de interrupción en diferentes anchos de pantalla.
  • tiny tweaks Permite realizar pequeños cambios en el diseño, como ajustar el tamaño de la funte, cambiar el tamaño de las imágenes o desplazar o el contenido de una manera no tan significativa. trabaja correctamente con diseñes de una sola columna.
  • off canvas. Coloca el contenido menos usado fuera de la pantalla y solo lo muestra cuando el tamaño de la pantalla es sufrientemente grande.

Look&Feel

Es una expresión inglesa que puede tener varios significados dependiendo del contexto en el cual se use. Significa textualmente el “aspecto y tacto”, mientras que, en el contexto de desarrollo, su significado es “aspecto y comportamiento”.

El lool&feel de una GUI, es el conjunto de propiedades y características que le dan una identidad visual única y puede ser percibidos de manera diferente de acuerdo con cada usuario.

El aspecto o look de las interfaces se define por los siguientes factores:

  • Paleta de colores.
  • Imagenes.
  • Diseño.
  • Opciones de fuente.
  • Estilo general.
  • Disposición de elementos

Feel es el comportamiento de las interfaces esta influido por la interacción que se determina por las siguientes características

  • El movimiento y la respuesta de los componentes dinámicos y como menús despegables, botones, formas y galerías.
  • Los efectos de sonido.
  • La rapidez con la que las páginas y las imágenes se cargan.

El aspecto y el comportamiento de un proyecto web son dos cuestiones fundamentales, es importante realizar un análisis previo para establecer que queremos reflejar, ya que en un instante muestra una actitud a los usuarios.

También se puede describir como su “personalidad”. La cual debe coincidir con la actitud de la empresa y sus objetivos. Y aunque existe una cierta libertar de diseño, los usuarios pueden confundirse por los proyectos web cuyo look&feel no cumple expectativas.

El look&feel puede ser descrito usando adjetivos. Por ejemplo:

  • Amistoso.
  • Accesible.
  • Profesional.
  • Experimentado.
  • Exclusivo.
  • Innovador.
  • Elegante.
  • Alta tecnología.
  • Poderoso.

Por otra parte, los proyectos web mal considerados pueden caer en categorías menos favorables:

  • Aburrido
  • Tenso
  • Pesado
  • Anticuado
  • Desordenado
  • Confuso
  • Infantil
